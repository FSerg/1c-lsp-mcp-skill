# Быстрый старт

1. Подготовка окружения:
    - Устанавливаем JVM. На Windows это можно одной командой: `winget install Microsoft.OpenJDK.25` или так `choco install openjdk`.
    - Скачиваем последний [релиз bsl-language-server](https://github.com/1c-syntax/bsl-language-server/releases) (проверялось все на v0.28.5)

2. Запуск приложения:
    - Запускаем серверную часть `lsp-skill-server`, web-интерфейс будет доступен по адресу http://127.0.0.1:4000 (позже можно будет установить приложение как службу)
    - Прописываем в PATH папку с CLI-утилитой, если хочется работать через SKILLs, это можно командой `../lsp-skill install-path`
3. Настройка приложения:
    - Указываем расположение файла `../bsl-language-server-0.28.5-exec.jar`
    - Включаем MCP, если необходимо (и перезапускаем сервер, если меняли эту настройку)
    - Добавляем проекты, указывая корневой путь всего проекта и путь к папке выгрузки исходников 1С (для правильного расчета путей для bsl-language-server)
    - Каждый проект
4. Работа с приложением:
    - Нажимаем "Запустить" у выбранного проекта. Это запустит прогрев/индексацию папок с выгрузками 1С (включая расширения). В терминах bsl-language-server это "наполнение контекста". На типовых УТ/БП/УНФ это несколько минут.
    - Далее в папке конкретного 1С-проекта настраиваете работу в одном из двух вариантов:
        - **Вариант 1: через SKILLs**. Для этого в корне проект должен быть файл `.env` с таким содержимым: `PROJECT_ID=55535b0f-886f-4dff-ad12-22377f9a5213`. И сами скилы либо в папке `.claude/skills/..`, либо `.agents/skills/..`
        
        - **Вариант 2: через MCP**. Настройки mcp у разных агентов немного отличаются, но обычно это `mcp.json` подобного вида: 
        ```
        {
            "servers": {
                "1c-lsp-diagnostics": {
                    "headers": {
                        "x-project-id": "fe9da3c9-5ce2-47cb-808c-acc6bb4b7b91"
                    },
                    "type": "http",
                    "url": "http://127.0.0.1:9011/mcp"
                },
                "1c-lsp-navigation": {
                    "headers": {
                        "x-project-id": "fe9da3c9-5ce2-47cb-808c-acc6bb4b7b91"
                    },
                    "type": "http",
                    "url": "http://127.0.0.1:9012/mcp",
                }
            },
            "inputs": []
        }
        ```
        где `PROJECT_ID` или `x-project-id` - это ID проиндексированного проекта.

Все, теперь LLM-агент может почти мгновенно делать проверку синтаксиса и навигацию по коду.

### Опции

Опционально, приложение можно установить как службу, чтобы не запускать его вручную каждый раз. Для этого есть встроенные команды:
- `../lsp-skill-server service install` - установка службы
- `../lsp-skill-server service uninstall` - удаление службы
- `../lsp-skill-server service status` - показать статус службы
